# cf-proxyip-us 修复记录

时间：2026-06-02

## 已完成的本地修复

- `worker.js`
  - `/health` 改为最小公开健康信息。
  - 新增需认证的 `/health/full`。
  - `/stats` 改为需要 Cookie/HMAC Token 认证。
  - 增加 stale 判断：6 小时标记 stale，12 小时 `ok: false`。
  - 支持 `HEAD` 请求。
  - 304 响应统一带安全 header。
  - 增加 CSP 与 Permissions-Policy。
  - Rate limiter 增加过期 bucket 清理。
  - 新增 `/base64.txt`，保留 `/v2ray.txt` 兼容旧链接。

- `build_dataset.py`
  - 修复直接 HTTPS fallback 检测结果与 `enrich()` 字段兼容问题。
  - 直接 HTTPS fallback 改为检查响应 header 中的 `cf-ray` / `server: cloudflare`。
  - 增加当前主 IP 质量门槛：bot score、corporate proxy、verified bot、延迟、目标地区。
  - 增加切换冷却时间，避免检测波动时频繁切换。
  - `docs/full.json` 输出不再包含完整 debug `all_results`。
  - 生成 `docs/base64.txt`，`docs/v2ray.txt` 保持兼容。

- `scripts/*.py`
  - `auto_update.py` 恢复 HMAC token 生成，不再使用纯日期 token。
  - `audit.py` 使用 `/health/full` 做详细健康验证。
  - `sync_dns.py` 支持 `PROXYIP_ZONE_NAME` / `PROXYIP_RECORD_NAME` 环境变量覆盖。
  - `auto_update.py` / `audit.py` 支持 `PROXYIP_LIST_DOMAIN` / `PROXYIP_RECORD_NAME` 环境变量覆盖。

- `README.md`
  - 移除已过期的动态统计数字。
  - 明确 `PROXYIP_HMAC_SECRET` 与 Cloudflare Worker Secret `PROXYIP_SECRET` 的关系。
  - 明确 `/health`、`/health/full`、`/stats` 的认证差异。
  - 补充 Cloudflare managed robots / AI Content Signals 可能接管 `/robots.txt` 的说明。

- `docs/`
  - `docs/full.json` 已移除 `all_results`，降低仓库体积。
  - 新增 `docs/base64.txt`。

## 已运行验证

```bash
python3 -m py_compile build_dataset.py scripts/*.py
node --check worker.js
```

- `best.txt` 是 `all.txt` 前 20 个。
- `top5.txt` 第一项是当前主 IP。
- `docs/base64.txt` 可解码，行数与 `all.txt` 一致。
